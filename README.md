# Vertical Video Feed

[![CI](https://github.com/Efremovtsev/vertical-video-feed/actions/workflows/ci.yml/badge.svg)](https://github.com/Efremovtsev/vertical-video-feed/actions/workflows/ci.yml)

Лента коротких видео с вертикальным листанием (в духе TikTok / Reels) на **чистом
JavaScript** - без фреймворков, без сборщика, без runtime-зависимостей. Работает
на мобильных и десктопе.

## Запуск

```bash
npm start            # статический сервер (npx serve .) → http://localhost:3000
```

Всё необходимое (видео, постеры, манифест) уже в репозитории - клонировал и
запустил. Пересобрать медиа из исходной папки Google Drive (нужны ffmpeg и gdown):

```bash
npm run build:feed
```

## Что умеет

- вертикальное листание с нативным `scroll-snap` - одно видео за жест;
- автозапуск активного видео через `IntersectionObserver`, старт всегда без звука
  (политика браузеров), глобальная кнопка звука наследуется следующими роликами;
- виртуализация: в DOM всегда 9 слайдов и **не больше 2 живых видеодекодеров**
  (активный + прогретый по направлению движения) - при любой длине ленты;
- ленивая постраничная подгрузка манифеста (бесконечный скролл);
- прогресс-бар, лайк (double-tap), пауза тапом, клавиатура (`↑`/`↓`, Space, M);
- состояния загрузки/ошибки с повтором, поддержка reduced-motion и bfcache.

## Почему решение именно такое

Задание оценивает структуру кода, отзывчивость и **эффективность использования
ресурсов браузера** - поэтому архитектура построена вокруг трёх дефицитных
ресурсов: DOM-узлы, аппаратные видеодекодеры (на мобильных их всего 1-2) и сеть.
Ключевые решения:

- **снап-точки живут на постоянной сетке пустых ячеек**, а не на переиспользуемых
  слайдах - быстрое листание никогда не теряет цель прилипания, и во время
  прокрутки не выполняется ни строчки JS;
- **«слайд в DOM» ≠ «слайд держит декодер»**: 9 узлов показывают постеры, видео
  получают только двое; при уходе слайда декодер и сетевая загрузка освобождаются
  явно (`pause → removeAttribute('src') → load()`);
- **при быстром листании декодирование не запускается вообще** - постеры
  «мостят» пролёт, декодер поднимается один раз в точке остановки.

Полное обоснование каждого решения - в **[ARCHITECTURE.md](ARCHITECTURE.md)**.

## Структура

```
index.html           оболочка: снап-контейнер, шаблон слайда, import map
styles.css           механика: снап, слои, композитные анимации
src/
  main.js            composition root - сборка и запуск
  FeedController.js  медиатор: единственный владелец сценария
  core/              EventBus, RafScheduler (единый rAF), viewport (измеренная H)
  engine/            виртуализация (не знает про видео), чистая оконная математика
  data/              постраничный манифест, лайки, контракт имён файлов
  media/             пул декодеров, политика воспроизведения
  view/              слайд и оверлей (пассивные вью)
  input/             жесты и клавиатура → семантические интенты
scripts/build-feed.mjs   Google Drive → ffmpeg → манифест (только на этапе сборки)
public/                  обработанные видео, постеры, JSON-манифест (в репозитории)
tests/                   юнит-тесты чистых модулей (node:test)
```

## Проверки качества

```bash
npm test             # 23 юнит-теста (node:test, без зависимостей)
npm run typecheck    # tsc --checkJs, strict - типы через JSDoc, без компиляции
npm run check        # Biome: линт + формат
```

Все три шага выполняются в CI на каждый push и в pre-commit хуке.

## Поддержка браузеров

Chrome / Edge / Firefox / Safari (десктоп и мобильные). Используются
`IntersectionObserver`, `scroll-snap`, `ResizeObserver`, `visualViewport`,
`scrollend` (с таймер-фолбэком там, где события нет).
